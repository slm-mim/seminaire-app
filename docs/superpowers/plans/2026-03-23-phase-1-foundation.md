# Phase 1 — Foundation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Set up the monorepo foundation with all tooling, configuration, and infrastructure so that feature development can begin immediately.

**Architecture:** Turborepo monorepo with pnpm workspaces. Two apps (`apps/api` for NestJS, `apps/web` for Next.js) and three shared packages (`packages/shared-types`, `packages/validation`, `packages/eslint-config`). Local development uses Docker Compose for PostgreSQL + Redis. CI runs on GitHub Actions.

**Tech Stack:** Turborepo, pnpm, NestJS, Next.js 14, TypeScript, Prisma, PostgreSQL, Redis, Tailwind CSS, shadcn/ui, Zod, ESLint, Prettier, Husky, lint-staged, Docker Compose, GitHub Actions

**Spec:** `docs/superpowers/specs/2026-03-23-seminaire-app-design.md`

  id                   String        @id @default(uuid())

  description          String
  speaker              String
  price                Decimal       @db.Decimal(10, 2)
  date                 DateTime
  location             String
  image                String?
  registrationDeadline Int

  creator       User            @relation(fields: [createdBy], references: [id])
  registrations Registration[]
  qaSession     QASession?
  emailCampaigns EmailCampaign[]
  driveSyncs    DriveSync[]

  @@map("seminars")
}

  id        String        @id @default(uuid())
  email     String        @unique
  firstName String
  lastName  String
  city      String
  phone     String?
  brevoId   String?
  source    ContactSource @default(MANUAL)
  createdAt DateTime      @default(now())
  updatedAt DateTime      @updatedAt

  registrations Registration[]

}

  contactId    String

  seminar Seminar @relation(fields: [seminarId], references: [id], onDelete: Cascade)
  contact Contact @relation(fields: [contactId], references: [id])

Seminar management application for non-technical organizers. Handles the full lifecycle: creation, registration, emailing (via Brevo), check-in, live Q&A, and post-event follow-up with Google Drive sync.

make setup    # Install deps, generate Prisma client, run migrations, build
